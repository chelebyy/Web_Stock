Mevcut durumda 2 rol düşünmüşsünüz (yönetici ve kullanıcı), bu yaklaşım birçok stok yönetim sistemi için temel seviyede yeterli olabilir. Ancak projenin kapsamına ve karmaşıklığına bağlı olarak ek roller düşünülebilir.

## Mevcut Roller
- **Yönetici (Admin)**: Tam sistem erişimi, kullanıcı yönetimi, raporlama yetkileri
- **Kullanıcı (User)**: Temel stok işlemleri, kendi işlemlerini görüntüleme

## Olası Ek Roller

Projenizin özelliklerine bağlı olarak şu rolleri ekleyebilirsiniz:

1. **Depo Sorumlusu (Warehouse Manager)**
   - Stok giriş/çıkışlarını onaylama
   - Stok transferlerini yönetme
   - Envanter sayımlarını gerçekleştirme

2. **Satın Alma Sorumlusu (Purchasing Officer)**
   - Sadece satın alma işlemlerine erişim
   - Tedarikçi yönetimi
   - Sipariş oluşturma ve takibi

3. **Rapor Görüntüleyici (Report Viewer)**
   - Sadece raporları görüntüleme yetkisi
   - Veri değiştirme yetkisi olmadan izleme

4. **Denetçi (Auditor)**
   - Salt okunur erişim
   - Tüm işlem geçmişine erişim
   - Hiçbir değişiklik yapamama

## Değerlendirme

İki rol yapısı (yönetici ve kullanıcı) aşağıdaki durumlarda yeterli olabilir:

- Küçük ölçekli bir işletme/organizasyon
- Sistemi kullanacak kişi sayısı az
- İş süreçleri karmaşık değil
- Yetki sınırlaması çok önemli değil

Daha fazla role ihtiyaç duyabileceğiniz durumlar:

- Kuruluşunuz büyüdükçe
- İş süreçleri daha karmaşık hale geldikçe
- Farklı departmanlar farklı sorumluluklarla sistemi kullandıkça
- Veri güvenliği ve erişim kontrolü daha kritik hale geldikçe

## Önerim

Şu an başlangıç için iki rol yeterli olabilir, ancak sisteminize:

1. **Rol bazlı yetkilendirme altyapısını esnek tasarlayın** - ileride kolayca yeni roller ekleyebilecek şekilde
2. **Detaylı yetki mekanizması ekleyin** - her rolün tam olarak hangi sayfalara ve işlevlere erişebileceğini belirleyen

Bu şekilde, sistem büyüdükçe veya ihtiyaçlar değiştikçe rol yapısını genişletmeniz kolaylaşacaktır.

Siz projeniz için hangi özellikleri öncelikli görüyorsunuz? Bu da rol ihtiyaçlarını belirlemede yardımcı olabilir.

_****_

```diff
- 
+ using Microsoft.AspNetCore.Mvc;
+ using Microsoft.AspNetCore.Mvc.Filters;
+ using Stock.Application.Common.Interfaces;
+ using System;
+ using System.Linq;
+ using System.Security.Claims;
+ using System.Threading.Tasks;
+ 
+ namespace Stock.API.Attributes
+ {
+     [AttributeUsage(AttributeTargets.Class | AttributeTargets.Method, AllowMultiple = true)]
+     public class RequirePermissionAttribute : TypeFilterAttribute
+     {
+         public RequirePermissionAttribute(string permissionName) : base(typeof(RequirePermissionFilter))
+         {
+             Arguments = new object[] { permissionName };
+         }
+     }
+ 
+     public class RequirePermissionFilter : IAsyncAuthorizationFilter
+     {
+         private readonly string _permissionName;
+         private readonly IPermissionService _permissionService;
+ 
+         public RequirePermissionFilter(string permissionName, IPermissionService permissionService)
+         {
+             _permissionName = permissionName;
+             _permissionService = permissionService;
+         }
+ 
+         public async Task OnAuthorizationAsync(AuthorizationFilterContext context)
+         {
+             // Kullanıcı kimliği doğrulanmamışsa
+             if (!context.HttpContext.User.Identity.IsAuthenticated)
+             {
+                 context.Result = new UnauthorizedResult();
+                 return;
+             }
+ 
+             // Admin rolü varsa her zaman izin ver
+             if (context.HttpContext.User.IsInRole("Admin"))
+             {
+                 return;
+             }
+ 
+             // Token'daki izinleri kontrol et
+             var hasPermission = context.HttpContext.User.Claims
+                 .Where(c => c.Type == "Permission")
+                 .Any(c => c.Value == _permissionName);
+ 
+             if (hasPermission)
+             {
+                 return;
+             }
+ 
+             // Veritabanından izinleri kontrol et
+             var userIdClaim = context.HttpContext.User.FindFirst(ClaimTypes.NameIdentifier);
+             if (userIdClaim != null && int.TryParse(userIdClaim.Value, out int userId))
+             {
+                 hasPermission = await _permissionService.HasPermissionAsync(userId, _permissionName);
+                 if (hasPermission)
+                 {
+                     return;
+                 }
+             }
+ 
+             // İzin yoksa 403 Forbidden hatası döndür
+             context.Result = new ForbidResult();
+         }
+     }
+ }
```
